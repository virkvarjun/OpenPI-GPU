The premise

openpi trains on one host. Real robot data does not fit that budget.

openpi is Physical Intelligence's open-source stack for π-family vision-language-action models. Its training script runs on a single host, and the codebase explicitly guards against multi-process use. Fine-tuning the full model already needs an 80 GB accelerator; scaling to more data means scaling across devices and hosts.

Sharder adds that path. The design has two non-negotiables: the single-host path stays byte-for-byte identical to upstream, and no optimization ships until the bottleneck it targets has been measured on real hardware.

Design constraints

  • Additive: single host is unchanged and untouched.
  • Deterministic: multi-process loss and gradients match single-process exactly.
  • Measure first: attribution instrument before any tuning.
  • Fault-tolerant: exact resume and elastic restart.
  • Validated on hardware, not simulated.
The study

A measurement pipeline, run end to end.

Each stage produces a number that decides the next. Nothing is optimized on a hunch.

Instrument the step

Attribute device time per HLO category and compute MFU from the analytical FLOP count. Two profiling errors are corrected up front.

Find the single-GPU ceiling

Establish where the time actually goes on one H100, and rule out the speculative optimizations that the numbers do not support.

Scale out with FSDP

Shard parameters across devices and measure strong and weak scaling on 1, 2, and 4 GPUs.

Attack the measured bottlenecks

FlashAttention for the attention cost that trace attribution hides; XLA overlap flags for the residual collective time.

Make it fault-tolerant

Exact deterministic resume from the step counter, and an elastic supervisor that restarts from the last checkpoint.

The internals

How the parallelism works.

FSDP: shard, gather, compute, scatter

Each device holds a slice of every weight matrix. Before a layer runs, the slices are all-gathered into the full matrix; after the backward pass, the gradients are reduce-scattered back to slices. Memory stays sharded; only the working matrix is ever whole.

The scaling behaviour below follows entirely from one question: can that gather hide behind the layer's compute?

All-gather assembles the full weight; the matmul runs; gradients reduce-scatter back to shards. Repeats every layer.
The naive path holds the full score matrix (grows with sequence squared); flash streams a tile and never stores it.

Attention: why the memory matters

Trace attribution reports attention near zero percent, because XLA fuses the masked softmax into the neighbouring einsum. Direct ablation shows the real cost, and that it scales with the square of sequence length. FlashAttention never materializes the score matrix, so it is faster and, more importantly, memory cheap enough to unlock long context.

Integrated behind a flag, on by default, numerically identical to the naive path for training and inference.

The headline finding: the single-GPU step is already compute-bound and GEMM-saturated, so there is no free lunch in the kernel. Throughput comes from parallelism, and the parallel efficiency is governed by whether the FSDP collective can hide behind compute. Weak scaling keeps each device fed and recovers most of the ideal.
Impact

What this means for openpi going forward.

A multi-host training path that does not break the old one

openpi gains sharded, multi-device training with deterministic parity to the single-host result, plus exact resume and elastic restart. The single-host path is unchanged, so it is a safe, additive contribution rather than a rewrite.

FlashAttention as a default, and a lever for long context

A free 9.3% today, and the enabling change for longer-horizon, more-camera policies where attention grows toward the majority of the step. Verified bit-identical, so it ships on by default with no retraining.

A reusable measurement culture

The attribution instrument, MFU harness, and scaling profilers outlive this study. They turn "should we optimize X" into a number, and they already retired several tempting-but-wrong ideas before any effort went into them.

A clear path to production scale

The remaining work is scoped and named: a healthy multi-node NCCL environment, fine-grained RLDS resume, and a full convergence run on real data. The systems groundwork for each is already in place.
