Sharder: scaling openpi training one measured bottleneck at a time.

Sharder adds a fault-tolerant, multi-host JAX training path to Physical Intelligence's openpi. The method is deliberately conservative: an optimization is applied only after the bottleneck it targets has been measured on real hardware, and the single-host path is left byte-for-byte identical to upstream. The work was validated on a four-GPU H100 node.

Premise

openpi trains on a single host

openpi is Physical Intelligence's open-source stack for the π-family of vision-language-action models. Its training script runs on one host and the codebase explicitly guards against multi-process use. Full fine-tuning already requires an eighty-gigabyte accelerator, so training on more data means training across more devices. Sharder supplies that path under two constraints: the single-host path stays unchanged, and no optimization ships before the bottleneck it targets is measured.1The discipline is not cosmetic. Three plausible optimizations, batched host-to-device transfer, an input-pipeline rewrite, and fused attention at the current sequence length, were each retired by a measurement before any effort went into them.

Method

The study

Each stage produces a number that determines the next. The pipeline runs from instrumentation to a validated scale-out result.

  1. Instrument the step

    Attribute device time per HLO category and compute MFU from the analytical FLOP count, correcting two profiling errors first.

  2. Find the single-GPU ceiling

    Establish where the time goes on one H100 and rule out the optimizations the numbers do not support.

  3. Scale out with FSDP

    Shard parameters across devices; measure strong and weak scaling on one, two, and four GPUs.

  4. Attack the measured bottlenecks

    FlashAttention for the attention cost that trace attribution hides; XLA overlap flags for the residual collective time.

  5. Make it fault-tolerant

    Exact deterministic resume from the step counter, and an elastic supervisor that restarts from the last checkpoint.

Internals

How the parallelism works

Under fully-sharded data parallelism each device holds a slice of every weight matrix. Before a layer runs, the slices are all-gathered into the full matrix; after the backward pass the gradients are reduce-scattered back into slices. Parameter memory stays sharded and only the working matrix is ever whole. Whether that gather can hide behind the layer's compute is the single fact that governs the scaling results below.

Attention is the second internal worth drawing. Trace-based attribution reports it near zero percent, because XLA fuses the masked softmax into the neighbouring einsum and charges the fused time to the matmul category. The naive path materializes the full score matrix, whose size grows with the square of the sequence length; FlashAttention streams a tile and never stores it, which is what makes it both faster and cheap enough in memory to unlock long context.

Results

Measurements on four H100s

The single-GPU step is compute-bound and GEMM-saturated at thirty-two percent MFU, so there is no headroom in the kernel itself. Throughput therefore has to come from parallelism, and the parallel efficiency is set by whether the FSDP collective overlaps compute.

Under strong scaling the per-GPU batch shrinks to one sample at four GPUs, the parameter all-gather dominates, and per-GPU MFU falls to 12.5 percent. Weak scaling keeps each device fed and reaches 2.85 times aggregate throughput on four GPUs, 980 TFLOP/s, at 24.8 percent MFU.

FlashAttention is wired into gemma under a flag that is on by default, and is numerically equivalent to the naive path for both training and inference, with loss parity to bit precision. End to end on gemma_2b it is 9.3 percent faster, 195.9 to 177.6 milliseconds, MFU 34.7 to 38.3 percent. The reduction exceeds the forward-only ablation because the backward pass benefits more: the naive path stores and differentiates the full score matrix, while the fused kernel recomputes it.2Two limitations. The cuDNN flash kernel rejects gemma_2b's head dimension of 256, which is above its limit of 128, so the XLA implementation is used. PagedAttention is specific to autoregressive inference with a KV cache and does not apply to the training step.

The residual weak-scaling gap is the portion of the collective that is not overlapped with compute. A tuned set of XLA flags, latency-hiding scheduler with pipelined async collectives and large combine thresholds, recovers 2.3 percent at two GPUs and 3.9 percent at four. The gain is small and grows with device count; recent jaxlib already overlaps most collectives by default, so the remainder is largely the fraction of the collective that cannot be hidden rather than a tuning deficiency.

The headline finding is negative in the useful sense: there is no free lunch in the kernel. Once the step is GEMM-saturated, throughput is a scaling problem, and scaling efficiency is a communication problem. Weak scaling is the lever that keeps each device fed.

Impact

What this means for openpi

A multi-host path that does not break the single-host one. openpi gains sharded, multi-device training with deterministic parity to the single-process result, plus exact resume and elastic restart. Because the single-host path is untouched, this is an additive contribution rather than a rewrite.

FlashAttention as a default, and a lever for long context. A free 9.3 percent today, and the enabling change for longer-horizon, more-camera policies where attention grows toward the majority of the step. It is verified bit-identical, so it ships on by default with no retraining.

A reusable measurement culture. The attribution instrument, the MFU harness, and the scaling profilers outlive this study. They convert "should we optimize X" into a number and already retired several tempting but wrong ideas before any effort went into them.

A scoped path to production scale. The remaining work is named: a healthy multi-node NCCL environment, fine-grained RLDS resume, and a full convergence run on real data. The systems groundwork for each is in place.
